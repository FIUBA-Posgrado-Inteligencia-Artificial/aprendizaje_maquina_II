# Glosario inicial del curso

Este archivo es el punto de partida para el glosario colaborativo que los alumnos
construyen durante la cursada en Moodle. Se pre-cargan los términos esenciales
para la Clase 1 como referencia.

---

| Término | Definición | Clase donde aparece |
|---|---|---|
| MLOps | Conjunto de prácticas que combina ML, DevOps e ingeniería de datos para estandarizar y automatizar el ciclo de vida de modelos en producción. | 1 |
| Pipeline de ML | Secuencia reproducible de pasos (ingestión, transformación, entrenamiento, evaluación, predicción) que produce o aplica un modelo. | 1 |
| Artefacto | Cualquier archivo o dato producido y almacenado durante el pipeline (modelo serializado, dataset procesado, reporte de métricas). | 1 |
| Reproducibilidad | Capacidad de obtener los mismos resultados ejecutando el mismo código con los mismos datos. | 1 |
| Drift | Cambio en la distribución de los datos de entrada o en la relación entre entrada y salida que degrada la performance del modelo en producción. | 8 |
| Model Registry | Sistema que versiona y cataloga modelos entrenados, permitiendo promoverlos entre etapas (Staging → Production). | 4 |
| Batch prediction | Patrón de despliegue donde el modelo genera predicciones sobre un conjunto de datos completo a intervalos regulares, en contraposición al serving online. | 7 |
| CI/CD | Integración Continua / Entrega Continua: prácticas de automatizar la validación y el despliegue del código ante cada cambio. | 2 |
